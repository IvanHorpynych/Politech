Modelli iAS
Replica di viste materializzate>Modelli iAS
Un modello iAS contiene un set di oggetti di cui verrà creata un'istanza su un database di livello intermedio. È possibile memorizzare nelle applicazioni iAS gli oggetti ai quali si accede spesso su un database di livello intermedio per migliorare le prestazioni e la scalabilità delle applicazioni.
Per creare un modello iAS su un database backend, effettuare le operazioni riportate di seguito.
  1. Creare un utente proprietario del modello che disponga di privilegi CONNECT e del ruolo SELECT_CATALOG_ROLE (oppure dei privilegi DBA). Eseguire quindi la connessione come proprietario del modello.
  2. Creare un modello iAS che definisca gli oggetti da creare sui siti di livello intermedio.
  3. Preparare il modello per l'esportazione per impostarlo in modo che sia possibile crearne le istanze.
  4. Esportare il modello per salvarne la definizione in un file di dump di esportazione.
  5. Copiare il file di dump di esportazione sui computer di livello intermedio sui quali si desidera creare le istanze del modello.
Per distribuire il modello ai siti di livello intermedio, effettuare le operazioni riportate di seguito.
  1. Eseguire la connessione al database di livello intermedio e creare un utente proprietario del modello dotato di privilegi DBA.
  2. Eseguire nuovamente la connessione al database di livello intermedio come proprietario del modello.
  3. Importare il modello per crearne un'istanza.
  4. Impostare i criteri di aggiornamento per il gruppo di aggiornamento associato al modello di cui si è appena creata un'istanza.
Una volta creata un'istanza del modello su tutti i database di livello intermedio, eseguire la connessione al database backend ed eseguire la funzione Cleanup dopo la creazione delle istanze.
