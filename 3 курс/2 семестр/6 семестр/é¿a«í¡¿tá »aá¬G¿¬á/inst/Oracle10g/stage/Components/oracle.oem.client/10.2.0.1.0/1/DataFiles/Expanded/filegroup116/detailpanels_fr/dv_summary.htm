Summary Management
Oracle prend en charge les vues matérialisées, également appelées récapitulatifs, pour améliorer les performances d'une base de données data warehouse. Ces récapitulatifs précalculent les jointures et les opérations d'agrégation onéreuses, puis les stockent dans une table. Oracle accélère l'exécution des interrogations en les récrivant de façon transparente pour utiliser des récapitulatifs. Les dimensions participent à cette opération de réécriture. Les journaux de vues matérialisées effectuent le suivi des modifications apportées à la table maîtresse pour la régénération incrémentielle des vues matérialisées.
Utilisez Dimensions pour définir des relations hiérarchiques entre des paires ou des ensembles de colonnes.
Utilisez Vues matérialisées pour améliorer les performances de la base de données data warehouse en stockant les jointures et les opérations d'agrégation précalculées.
Utilisez Journal des vues matérialisées pour effectuer le suivi des modifications apportées à la table maîtresse, en vue de la régénération incrémentielle des vues matérialisées.
Utilisez Groupe de régénération afin de conserver la cohérence transactionnelle existant entre les vue matérialisées.
Utilisez Summary Advisor pour obtenir des conseils concernant la création de vues matérialisées destinées à la réécriture des interrogations, afin d'améliorer les performances des applications de la base de données data warehouse.
