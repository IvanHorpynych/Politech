Tipi di origine
Per lo sviluppo della logica di applicazione nel database, Oracle supporta oggetti che costituiscono unità di codice scritto in linguaggio SQL, PL/SQL, Java o C.
Utilizzare Package per creare o sfogliare un gruppo di procedure e funzioni correlate memorizzate come singola unità.
Utilizzare Package body per creare o sfogliare tutti i costrutti pubblici e privati del package. È possibile compilare il package body ed eseguirne il debug.
Utilizzare Procedura per sviluppare codice con SQL e PL/SQL.
Utilizzare Funzione per sviluppare codice che restituisce sempre un singolo valore al chiamante.
Utilizzare Trigger per sviluppare codice che viene eseguito quando una tabella o una vista viene modificata oppure quando si verificano determinate azioni utente o del sistema di database.
Utilizzare Origine Java per compilare codice Java per un set di bytecode standard e indipendente dalla piattaforma che interagisce con una specifica Java Virtual Machine (JVM).
Caricare una classe Java nel database per un'esecuzione che preveda l'utilizzo della specifica Java Virtual Machine (JVM) nel database.
SUGGERIMENTO Per le operazioni di amministrazione e diagnostica, la Oracle consiglia di utilizzare Enterprise Manager 10g Console.
