Gestione riepiloghi
Per migliorare le prestazioni di un data warehouse, Oracle supporta le viste materializzate, definite anche riepiloghi. I riepiloghi calcolano in anticipo le operazioni di join e aggregazione costose e le memorizzano in una tabella. Oracle rende più veloce l'esecuzione delle query riscrivendo in modo trasparente le query per l'utilizzo dei riepiloghi. A loro volta, le dimensioni rendono più veloce l'operazione di riscrittura delle query. I log MV tengono traccia delle modifiche apportate alla tabella principale per definire l'aggiornamento incrementale delle viste materializzate.
Utilizzare le dimensioni per definire le relazioni gerarchiche tra coppie di colonne o di set di colonne.
Utilizzare le viste materializzate per migliorare le prestazioni del warehouse tramite la memorizzazione di operazioni di join e di aggregazione calcolate in anticipo.
Utilizzare il log delle viste materializzate per tenere traccia delle modifiche apportate alla tabella principale per l'aggiornamento incrementale delle viste materializzate.
Utilizzare un gruppo di aggiornamento per mantenere coerenti le viste materializzate le une rispetto alle altre tra le transazioni.
Utilizzare Summary Advisor per ottenere consigli per la creazione di viste materializzate per la riscrittura delle query al fine di migliorare le prestazioni delle applicazioni del warehouse.
