Gestione di Streams
Oracle Streams consente di condividere in modo semplice e flessibile messaggi e dati tra database diversi. Ad esempio, è possibile utilizzarlo per recuperare come eventi le modifiche DML e DDL apportate agli oggetti del database. Successivamente questi eventi potranno essere propagati a un altro database per una replica efficiente degli oggetti del database su altri database.
In Oracle Streams sono previsti i tre processi principali riportati di seguito.
Il recupero consente di recuperare le modifiche apportate agli oggetti del database dai redo log. Queste modifiche vengono inserite in una coda.
La propagazione consente di propagare le modifiche dalla coda del database di origine alle code dei database di destinazione.
L'applicazione consente di recuperare le modifiche dall'area della coda di destinazione e di applicarle al database.
Oracle Enterprise Manager Console consente di impostare Streams e di monitorare i processi Streams.
