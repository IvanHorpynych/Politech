Concurrent Manager
 
Concurrent Manager planen die Ausführung von Verarbeitungsanforderungen in Oracle Applications-Datenbanken. Mit der Konsole können Sie:
Ereignisse erstellen, um Concurrent Manager auf Bedingungen zu überwachen, die ein Eingreifen erfordern.
Stapeljobs zur Behebung dieser Bedingungen planen.
Zeiträume für Benachrichtigungs-Blackouts festlegen oder einen Blackout für alle Benachrichtigungen für diesen Concurrent Manager bestimmen.
Capacity Planner zur Analyse historischer Trends und Projizierung künftiger Kapazitäten verwenden.
Perfomance Manager-Diagramme benutzen, um Concurrent Manager-Statistiken zu analysieren.
Berichte mit historischen Daten aus Concurrent Managern, Concurrent Programs und Concurrent Program-Verarbeitungen generieren.
Einen Concurrent Manager aus dem Navigatorbaum löschen.
