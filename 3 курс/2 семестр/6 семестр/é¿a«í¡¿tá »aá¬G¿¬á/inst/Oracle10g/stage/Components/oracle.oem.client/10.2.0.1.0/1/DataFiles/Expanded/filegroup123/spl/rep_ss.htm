Materialized View Replication allows complete or partial replicas of tables on the source database (called the Master site) to be created on one or more destination databases (called Materialized View sites).
Materialized View Replication can be accomplished in different ways. Click a link to see more information:
  • Use Deployment Templates to efficiently deploy and administer a widely distributed materialized view environment.
  • Use Materialized Views to deploy and administer a read-only materialized view environment.
  • Use Materialized View Groups to deploy and administer an updatable materialized view environment.
  • Use Refresh Groups to refresh multiple materialized views in one step.
To learn more about Materialized View Replication, click the Help button.
