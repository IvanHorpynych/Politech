XML 스키마 기반 객체
XML 데이터베이스 관리>XML 스키마 기반 객체
XML 스키마는 XML로 된 스키마 정의 언어로, 일치하는 XML 인스턴스 문서의 구조 및 의미를 설명하는 데 사용할 수 있습니다.
Oracle 9.2를 사용하여 다음 작업을 수행할 수 있습니다.
기존 XML 데이터베이스 리소스를 사용하거나 로컬에서 또는 인터넷을 통해 액세스할 수 있는 XML 데이터베이스 스키마 파일을 사용하여 데이터베이스에 XML 스키마를 등록할 수 있습니다.
XML 스키마를 기반으로 구조화된 저장 영역을 생성할 수 있습니다.
다음 중 한 가지 방법으로 이 작업을 수행할 수 있습니다.
XML 스키마에서 정의된 구조를 기반으로 테이블을 생성할 수 있습니다.
XML 스키마에서 정의된 구조를 기반으로 하는 열이 있는 테이블을 생성할 수 있습니다.
XML 스키마 기반 저장 영역을 생성하면 다음과 같은 이점이 있습니다.
  • 이 저장 영역에 저장된 XML 인스턴스 문서가 훨씬 적은 공간을 차지합니다.
  • XPath를 통해 표현되는 특정 데이터 조각에 대한 액세스가 상당히 최적화되었습니다.
  • XML 인스턴스 문서의 일부를 갱신하는 작업이 크게 향상되었습니다.
XML 스키마를 기반으로 뷰를 생성하여 기존 데이터를 XML 형식으로 바꿀 수 있습니다.
XPath 표현식을 사용하여 XML 스키마 기반 테이블에 함수 인덱스를 생성할 수도 있습니다.
