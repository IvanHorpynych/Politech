Summary Management
Oracle supports Materialized Views, also known as Summaries, to improve the performance of a data warehouse. Summaries pre-calculate expensive joins and aggregation operations and store them in a table. Oracle speeds-up query execution by transparently re-writing queries to use summaries. Dimensions aid query re-write operation. MV Logs track changes to master table for incremental refresh of Materialized Views.
Use Dimensions to defines hierarchical relationships between pairs of columns or column sets.
Use Materialized Views to improve the performance of your warehouse by storing pre-calculated joins and aggregate operations.
Use Materialized View Log to track changes to master table for incremental refresh of Materialized Views.
Use Refresh Group to keep materialized views transactionally consistent with each other.
Use Summary Advisor to get recommendations for the creation of materialized views for query rewrite to improve the performance your warehouse applications.
