Modèles iAS
Réplication de vues matérialisées>Modèles iAS
Un modèle iAS contient un ensemble d'objets qui seront instanciés sur une base de données de niveau intermédiaire. Les applications iAS peuvent stocker les objets fréquemment utilisés dans une base de données de niveau intermédiaire afin d'améliorer les performances et l'adaptabilité de l'application.
Pour créer un modèle iAS dans une base de données Backend :
  1. Créez un propriétaire de modèle doté de privilèges CONNECT et SELECT_CATALOG_ROLE (ou privilèges DBA), puis connectez-vous sous le nom utilisateur de ce propriétaire de modèle.
  2. Créez un modèle iAS qui définit les objets à créer sur les sites de niveau intermédiaire.
  3. Préparez le modèle pour l'export afin de le configurer de manière à pouvoir l'instancier.
  4. Exportez le modèle pour enregistrer la définition de modèle dans un fichier d'export (vidage).
  5. Copiez le fichier de vidage d'export sur les ordinateurs de niveau intermédiaire où vous souhaitez instancier le modèle.
Pour déployer le modèle vers des sites de niveau intermédiaire :
  1. Connectez-vous à la base de données de niveau intermédiaire et créez un propriétaire de modèle doté de privilèges DBA.
  2. Reconnectez-vous à la base de données de niveau intermédiaire sous le nom de ce propriétaire de modèle.
  3. Importez le modèle pour l'instancier.
  4. Définissez la politique de régénération du groupe de régénération associé au modèle venant d'être instancié.
Une fois le modèle instancié sur toutes les bases de données de niveau intermédiaire, connectez-vous à la base de données Backend et exécutez un nettoyage après instanciation.

Pour plus d'informations sur les modèles iAS, cliquez sur le bouton Aide.
