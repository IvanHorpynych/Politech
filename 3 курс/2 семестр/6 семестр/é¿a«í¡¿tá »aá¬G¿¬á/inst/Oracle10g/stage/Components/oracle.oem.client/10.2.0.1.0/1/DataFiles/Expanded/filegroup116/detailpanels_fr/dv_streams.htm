Gestion avec Oracle Streams
Oracle Streams permet de partager facilement et avec une grande souplesse des messages et des données entre plusieurs bases de données. Par exemple, vous pouvez utiliser Oracle Streams pour capturer sous forme d'événements les modifications DML et DDL apportées aux objets de base de données. Vous pouvez ensuite propager vers d'autres bases ces événements répliquant de manière efficace les objets de base de données vers les autres bases.
Oracle Streams se compose de trois processus principaux :
La capture des modifications apportées aux objets de base de données des fichiers de journalisation. Ces modifications sont placées dans une file d'attente.
La propagation des modifications stockées dans la file d'attente de la base de données source vers la file d'attente des bases de données de destination.
L'extraction des modifications de la file d'attente de destination et leur application dans la base de données.
La console d'Oracle Enterprise Manager vous permet de configurer des flux de données et de surveiller des processus Oracle Streams.
