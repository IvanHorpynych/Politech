iAS Templates
Materialized View Replication>iAS Templates
An iAS Template contains a set of objects that will be instantiated on a middle tier database. iAS applications can store frequently accessed objects on a middle tier database to improve application performance and scalability.
To create an iAS template on a backend database:
  1. Create a template owner user which has CONNECT privileges and the SELECT_CATALOG_ROLE (or DBA privileges). Then connect as the template owner.
  2. Create an iAS Template which defines the objects to be created on the middle tier sites.
  3. Prepare the template for export to setup the template so that it can be instantiated.
  4. Export the template to save the template definition in an export dump file.
  5. Copy the export dump file to the middle tier machines on which you want to instantiate the template.
To deploy the template to middle tier sites:
  1. Connect to the middle tier database and create a template owner user which has DBA privileges.
  2. Reconnect to the middle tier database as the template owner.
  3. Import the template to instantiate the template.
  4. Set the refresh policy for the refresh group associated with the newly instantiated template.
Once you have instantiated the template on all the middle tier databases, connect to the backend database and perform Post Instantiation Cleanup.
