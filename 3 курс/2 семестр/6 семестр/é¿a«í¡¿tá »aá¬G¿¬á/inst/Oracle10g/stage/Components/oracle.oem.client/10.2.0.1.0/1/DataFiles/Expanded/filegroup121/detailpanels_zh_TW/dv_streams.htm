Streams Management
Oracle Streams provides a simple and flexible way to share messages and data between databases. For example, you can use Streams to capture DML and DDL changes to database objects as events. You can then propagate these events to other database effectively replicating the database objects to other databases.
Oracle Streams consists of three main processes:
Capture to capture changes to database objects from redo logs. These changes are placed in a queue.
Propagate to propagate changes from queue at source database to queue at destination databases.
Apply to retrieve changes from the destination queue area and apply to the database.
Oracle Enterprise Manager Console allows you to Setup Streams and monitor Streams processes.
