Gestión de Streams
Oracle Streams proporciona una forma sencilla y flexible de compartir mensajes y datos entre bases de datos. Por ejemplo, puede utilizar Streams para capturar cambios DML y DDL realizados en los objetos de base de datos como eventos. A continuación puede propagar estos eventos a la otra base de datos, replicando de forma efectiva los objetos de la base de datos en otras bases de datos.
Oracle Streams se compone de tres procesos principales:
Capturar: Para capturar cambios en los objetos de base de datos a partir de los redo logs. Estos cambios se colocan en una cola.
Propagar: Para propagar cambios desde la cola de la base de datos origen a la cola de las bases de datos destino.
Aplicar: Para recuperar cambios del área de la cola de destino y aplicarlos a la base de datos.
La Consola de Oracle Enterprise Manager permite configurar Streams y controlar procesos de Streams.
