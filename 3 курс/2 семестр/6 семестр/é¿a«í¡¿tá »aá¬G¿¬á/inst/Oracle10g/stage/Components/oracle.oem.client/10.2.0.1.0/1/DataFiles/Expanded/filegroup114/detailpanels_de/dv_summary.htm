Verwaltung der Zusammenfassung
Oracle unterstützt Materialized Views, die auch als Zusammenfassungen bezeichnet werden, zur Verbesserung der Performance eines Data Warehouses. In Zusammenfassungen werden aufwendige Joins und Aggregationsvorgänge im Voraus berechnet und in einer Tabelle gespeichert. Oracle beschleunigt die Ausführung von Abfragen, indem Abfragen transparent zur Benutzung von Zusammenfassungen neu geschrieben werden. Mit Dimensionen kann das Neuschreiben von Abfragen vereinfacht werden. Mit MV-Logs werden Änderungen an der Master-Tabelle protokolliert, um Materialized Views inkrementell zu aktualisieren.
Mit Dimensionen definieren Sie hierarchische Beziehungen zwischen Spaltenpaaren oder Spaltengruppen.
Mit Materialized Views verbessern Sie die Performance Ihres Warehouses, indem im Voraus berechnete Joins und Aggregationsvorgänge gespeichert werden.
Mit einem Materialized View-Log protokollieren Sie Änderungen an einer Master-Tabelle, um Materialized Views inkrementell zu aktualisieren.
Mit einer Aktualisieren-Gruppe stellen Sie sicher, dass Materialized Views transaktionell konsistent bleiben.
Mit einem Summary Advisor rufen Sie Empfehlungen für das Erstellen von Materialized Views für das Umschreiben von Abfragen auf, um die Performance Ihrer Warehouse-Anwendungen zu verbessern.
